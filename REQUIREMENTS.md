# MyWispr — Requirements Specification

A stripped-down macOS push-to-talk voice transcription tool inspired by WisprFlow. Intended for personal use and sharing with friends and family on macOS.

## Overview

Hold or double-tap a hotkey → speak → release → transcript is auto-pasted at the cursor. All audio and transcripts stored locally. No cloud dependency.

## Target Platform

- macOS (Apple Silicon and Intel)
- Distributed as a hand-rolled minimal `MyWispr.app` bundle that launches `bash run.sh` (generated by `scripts/build-app.sh`)
- Shared with non-technical users (installation must be simple)

---

## Features

### 1. Push-to-Talk Hotkey

- User can configure the trigger key in settings
- Two interaction modes (also configurable):
  - **Hold-down**: hold key to record, release to transcribe
  - **Double-tap**: double-tap key to start recording, double-tap again to stop
- Default hotkey: Right Option; users can change it in settings if it conflicts

### 2. Audio Recording

- Record to a local `.wav` file on every invocation
- Files stored in a configurable local directory (default: `~/Library/Application Support/MyWispr/audio/`)
- Named by timestamp: `YYYYMMDD-HHMMSS.wav`
- Audio is retained for automated cleanup (see §8); transcripts are kept indefinitely

### 3. Transcription Engine

- **Default shared-install model**: Whisper base (`ggml-model-whisper-base.bin`)
- **Preferred upgrade model**: Whisper turbo (`ggml-model-whisper-turbo.bin`)
- **Integration**: `pywhispercpp` (Python bindings for whisper.cpp)
  - Model discovery order:
    1. User-configured model path
    2. MyWispr-owned turbo model at `~/Library/Application Support/MyWispr/models/ggml-model-whisper-turbo.bin`
    3. MacWhisper turbo model at `~/Library/Application Support/MacWhisper/models/ggml-model-whisper-turbo.bin`
    4. MyWispr-owned base model at `~/Library/Application Support/MyWispr/models/ggml-model-whisper-base.bin`
    5. MacWhisper base model at `~/Library/Application Support/MacWhisper/models/ggml-model-whisper-base.bin`
    6. Model-needed state
  - If no model is available, the app enters a model-needed state and offers a user-facing path to install/download a local model or select an existing compatible model file
  - Transcription remains fully local after the model is available; first-run model acquisition may require a bundled base-model asset, separate installer, or download flow
  - Model path remains user-overridable in settings
- **Language**: user-selectable with auto-detect as the default

**Rationale for model-source behavior:** The original plan assumed MacWhisper was installed and that MyWispr could reuse its downloaded Whisper turbo model. That is valid on the developer machine but not for friends/family who may not have MacWhisper. MyWispr must therefore treat MacWhisper reuse as an optimization, not a requirement. For v1 shared installs, Whisper base is the default because it is much smaller and makes the app easier to distribute; Whisper turbo remains the preferred quality upgrade when already present or installed later.

### 4. Auto-Paste

- On transcription complete: save to SQLite, copy to clipboard, then attempt Cmd+V via `osascript` / `System Events` — in that order
- Transcript and clipboard are always persisted before paste is attempted; a paste failure never loses the text
- Requires macOS Accessibility permission (granted to the app wrapper)

### 5. Clipboard Fallback

- The most recent transcript is always on the clipboard
- Menu bar dropdown shows the last N transcripts (default: 20) with a "Copy" option for each
- Allows retrieval if the user accidentally clicked away before the paste landed

### 6. Disfluency Removal

- Post-processing pass that strips common speech disfluencies before pasting
- Default removal list: `um`, `uh`, `like`, `you know`, `so`, `actually`, `basically`, `literally`, `I mean`, `right`
- User can add or remove words from the list in settings
- Removal should be conservative by default: only standalone filler patterns should be stripped, and legitimate uses of words like `so`, `right`, `actually`, and `like` should be preserved where possible
- LLM cleanup pass is explicitly out of scope for v1 (see Out of Scope)

### 7. Menu Bar App

- Lives in the macOS menu bar (ideally no Dock icon)
- Status indicator changes by state:
  - **Idle**: default icon
  - **Recording**: animated icon
  - **Processing**: distinct icon
  - **Permission needed**: distinct state shown when Microphone or Accessibility permission is missing, with a menu item linking to System Settings
  - **Model needed**: distinct state shown when neither the MacWhisper model nor the MyWispr-owned model is available, with menu items to install/download or choose a model file
- App checks for required permissions at launch and shows the appropriate degraded state before any recording or paste failure can occur
- App checks for model availability at launch before recording can begin
- Permission checks may use different implementation paths: Accessibility can be checked with macOS Accessibility APIs or a controlled System Events probe; Microphone may require an actual audio permission-triggering check rather than a simple boolean
- Dropdown menu includes:
  - Recent transcripts (last X) with ability to copy
  - Search Transcripts
  - Settings
  - Quit

### 8. Transcript History & Search

- All transcripts stored in a local SQLite database
  - `~/Library/Application Support/MyWispr/transcripts.db`
  - Schema: `id`, `timestamp`, `duration_sec`, `language`, `audio_file` (nullable), `raw_text`, `cleaned_text`
- Transcripts are **never** automatically deleted
- Simple v1 search UI filters by keyword and date range.

### 9. Automated Audio Cleanup

- Cleanup runs at app launch (opportunistic) and daily via a launchd timer
- Deletes `.wav` files older than a configurable threshold (default: 30 days)
- Transcripts are unaffected; the `audio_file` column is set to `NULL` after cleanup
- User can disable auto-cleanup or adjust the retention period in settings
- App-launch cleanup ensures users get reliable baseline behavior even if the launchd job is not installed, disabled, or misbehaving

### 10. Settings

Accessible from the menu bar via `rumps` dialogs and menu flows (no WebKit window). Persisted as JSON at `~/Library/Application Support/MyWispr/settings.json`.

| Setting | Default | Notes |
|---|---|---|
| Hotkey | Right Option | Any single key |
| Interaction mode | Hold-down | Hold-down or Double-tap |
| Language | Auto-detect | Any Whisper-supported language |
| Model quality / path | Auto (turbo if available, otherwise base) | Advanced users may choose a compatible whisper.cpp model file |
| Disfluency list | (see §6) | Editable list |
| LLM cleanup | — | Post-v1; not in settings for v1 |
| Audio retention | 30 days | Days before audio deleted |
| Recent transcripts shown | 20 | Menu bar history count |

## Technical Stack

| Component | Library / Tool |
|---|---|
| Menu bar UI | `rumps` |
| Global hotkey | `pynput` |
| Audio recording | `sounddevice` |
| Transcription | `pywhispercpp` (whisper.cpp Python bindings) |
| Auto-paste | `osascript` via `subprocess` |
| Clipboard | `pyperclip` |
| Transcript storage | SQLite (`sqlite3`, stdlib) |
| Background cleanup | `launchd` plist |
| Packaging / app wrapper | Hand-rolled `MyWispr.app` bundle calling `bash run.sh` (`scripts/build-app.sh`) |
| Python environment | Local venv or packaged runtime invoked by `run.sh` |

Model distribution is a separate packaging concern because Whisper turbo is large. For v1, the main shared-install path should use the smaller Whisper base model by default and treat turbo as an optional quality upgrade. MyWispr must not require MacWhisper to be installed; it should either include/provision the base model, provide a first-run local model install/download flow, or be distributed with a separate model installer package. Turbo should not be bundled in the main app package for v1 because it would make distribution too large.

## macOS Permissions

Three permissions required, all granted once around first launch/first use:

- **Microphone** — for audio recording (system prompt at first launch)
- **Accessibility** — for simulating Cmd+V via System Events (added manually in System Settings)
- **Automation → System Events** — for `osascript` control of System Events (system prompt on first paste; cannot be pre-granted)

Permission attribution follows the app-wrapper bash-indirection pattern:

- `MyWispr.app` is a hand-rolled minimal bundle: `Contents/MacOS/MyWispr` is a precompiled Mach-O trampoline (universal arm64+x86_64) that `execv`s `bash run.sh` → which invokes Python. `Info.plist` sets `LSUIElement` true (no Dock icon), `LSRequiresNativeExecution` true, and a stable `CFBundleIdentifier`.
- The bundle executable must call `bash run.sh`, never Python directly — otherwise TCC attributes prompts to Python instead of MyWispr.
- User grants the permissions to `MyWispr.app` in System Settings → Privacy & Security; TCC prompts show "MyWispr would like to…" rather than a Python process name.
- The launchd cleanup plist invokes `bash run.sh --cleanup-only` directly (an app launch cannot carry the flag); this is safe because cleanup only touches `~/Library/Application Support`, which is not TCC-protected — but the install root itself must be outside `~/Documents`, or launchd's bash access to the script trips the TCC Documents prompt.
- The shipped v1 artifact is the bundle plus its local Python runtime/dependencies and support files at `~/Library/Application Support/MyWispr/app/`; `py2app` is not part of v1 packaging.

**Rationale for packaging decision:** the wrapper must be the single shipped app identity that users grant permissions to, while `run.sh` controls the Python entrypoint underneath. The original v1 M1–M6 used an Automator Application for this (the previously documented pattern), but Automator displays a permanent gear/progress menu item while the workflow runs, and its ⓧ button terminates the app. The hand-rolled bundle keeps the identical TCC mechanics without that artifact, requires no Xcode or signing for personal distribution, and was adopted at M7. Do not reintroduce the Automator wrapper or `py2app` for v1.

## Out of Scope (v1)

- Voice Activity Detection (VAD) — auto-stop on silence
- System notification on transcription complete (auto-paste is the signal)
- Speaker diarization
- Export in multiple formats
- Onboarding wizard
- Windows / Linux support
- Cloud transcription fallback
- LLM cleanup pass (deterministic disfluency removal is sufficient for v1)
- Requiring MacWhisper to be installed
- `py2app` packaging

---

## Post-v1 Feature Requests

- **Custom vocabulary / terminology**: let users specify names, technical terms, or phrases that Whisper frequently mishears. Whisper supports an `initial_prompt` parameter that can bias transcription toward specific words — a simple editable text field in Settings could feed into this.
- **One-time setup help page**: a static local HTML page (opened in the browser) that walks through the three TCC grants with screenshots, so non-technical users have a persistent reference.
