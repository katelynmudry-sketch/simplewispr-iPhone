# MyWispr — Project Instructions

macOS push-to-talk voice transcription app. Inspired by WisprFlow. Python-based, shipped as a hand-rolled minimal `.app` bundle wrapper for v1.

## What this project is

- Hold or double-tap a hotkey → speak → release → transcript auto-pasted at cursor
- All local: no cloud dependency, no NVIDIA GPU required
- Intended for the user and non-technical friends/family on macOS (Apple Silicon and Intel)

## Key technical decisions already made

- **Transcription**: `pywhispercpp` using local whisper.cpp models only. `REQUIREMENTS.md` is the source of truth for model defaults, discovery order, and distribution rationale.
- **Model guardrails**: Do not require MacWhisper to be installed. Do not add NVIDIA Parakeet v3 support under `pywhispercpp`; the available Parakeet files are CoreML `.mlmodelc` bundles, not compatible whisper.cpp models.
- **Menu bar**: `rumps` library (no Dock icon)
- **Hotkey capture**: `pynput`
- **Audio recording**: `sounddevice` → local `.wav` files
- **Auto-paste**: `osascript` / System Events (simulates Cmd+V)
- **Storage**: SQLite (`~/Library/Application Support/MyWispr/transcripts.db`)
- **Distribution**: hand-rolled minimal `MyWispr.app` bundle — `Contents/MacOS/MyWispr` is a pre-compiled Mach-O trampoline (`scripts/trampoline`, universal arm64+x86_64, source in `scripts/trampoline.c`) that `execv`s `/bin/bash run.sh`. LaunchServices on macOS 15 ignores bash-script executables. `Info.plist` has `LSUIElement` true, `LSRequiresNativeExecution` true (forces arm64 on Apple Silicon — without it LaunchServices picks x86_64, mismatching the venv), and a stable `CFBundleIdentifier`. Generated by `scripts/build-app.sh`, which copies the trampoline binary and ad-hoc codesigns. Do not use `py2app` for v1. Do not use an Automator Application (superseded at M7). Do not replace the trampoline with a bash script.
- **Install root vs dev tree**: The app runs source files from `~/Library/Application Support/MyWispr/app/`, not from the dev tree. To see changes: `cp` the edited file to the install root (e.g. `cp src/main.py ~/Library/Application\ Support/MyWispr/app/src/main.py`), then restart the app. That's it — no cache clearing, no rebuild needed. `scripts/build-app.sh` only rebuilds the `.app` bundle skeleton, not source files.
- **macOS TCC permissions**: The exec chain is `trampoline → exec bash → exec python`. All three `exec` calls are required: trampoline satisfies LaunchServices' Mach-O requirement; bash preserves the responsible-process attribution so TCC grants go to `MyWispr.app`; `exec python` in run.sh ensures the LaunchServices-tracked PID chains to the NSApplication (satisfying RunningBoard's check-in). `run.sh` must use `exec python`, NOT bare `python`. Grants needed: Microphone, Accessibility (manual), and Automation → System Events (prompted on first paste). The install root must be outside `~/Documents` to avoid TCC Documents prompts from launchd.
- **pynput macOS 15**: `TISCopyCurrentKeyboardInputSource` now requires the main queue; pynput calls it from its listener thread. Fixed by `_patch_pynput_keycode_context()` in `main.py` — must be called from `main()` before `MyWisprApp().run()`. Do not remove this patch.
- **Dialog activation (M8)**: `activateIgnoringOtherApps_` is unreliable for LSUIElement apps on macOS 14/15. All alerts and windows use `_alert()`/`_window()` wrappers in `main.py` that set `NSFloatingWindowLevel` on the NSAlert's window before `runModal`. `rumps.Window` stores its NSAlert as `self._alert` (available before `.run()`). Do not revert to bare `rumps.alert()` calls.
- **UI-sync timer and modal sessions (M8)**: The rumps Timer's NSTimer fires in `NSDefaultRunLoopMode` only. After `self._timer.start()`, the timer is also added to `NSRunLoopCommonModes` via `NSRunLoop.currentRunLoop().addTimer_forMode_(self._timer._nstimer, NSRunLoopCommonModes)` so it keeps firing during modal sessions (alerts/windows). The private attribute is `_nstimer`. Do not remove this line.

## Features included in v1

1. Push-to-talk: hold-down or double-tap (configurable)
2. Local WAV recording
3. Transcription via pywhispercpp using local whisper.cpp models
4. Save to SQLite → copy to clipboard → auto-paste (in that order; paste failure never loses the text)
5. Clipboard fallback + recent transcript history in menu
6. Disfluency removal pass (configurable word list; LLM cleanup is out of scope for v1)
7. Menu bar with idle/recording/processing/permission-needed/model-needed states; permission and model preflight at launch
8. Language selection with auto-detect
9. SQLite transcript history with search
10. Audio cleanup at app launch + daily via launchd (default: delete WAVs older than 30 days; transcripts never deleted)
11. Settings via rumps dialogs (no WebKit window); persisted as JSON

## Features explicitly deferred (do not add unless asked)

- VAD (voice activity detection)
- System notification on transcription complete
- Speaker diarization
- Multi-format export
- Onboarding wizard
- Cloud transcription fallback
- Requiring MacWhisper to be installed
- `py2app` packaging for v1
- Automator wrapper (superseded by the hand-rolled bundle at M7 — do not reintroduce)
